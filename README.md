# AI英会話練習

英会話アプリを安全にデプロイし、公開前にセキュリティリスクを確認・修正するための再利用可能なスキル。

## 概要

このスキルは、英会話アプリのデプロイプロセスをサポートし、本番公開前に以下を確認・修正します：

- セキュリティリスク（シークレット漏洩、危険なコードパターン）
- 環境設定（APIキー、環境変数の正確性）
- CORS設定、CSP、入力バリデーション
- 外部依存関係の脆弱性確認
- 本番環境対応（ホスティング先の確認）

## 機能

### 1. 目的の確認
- デプロイ対象のアプリ構成（フロントエンド、バックエンド、API連携）
- 公開先の環境（Cloudflare Pages/Workers、Vercel、Netlify、AWS など）

### 2. 環境とシークレットの検証
- APIキーやシークレットのハードコード化チェック
- 環境変数・シークレット管理への登録確認
- 開発専用設定の本番環境への残留チェック

### 3. セキュリティリスクの確認
- 平文シークレット、`.env` ファイルなどの漏洩リスク
- `eval`、`new Function` などの危険パターン
- CORS設定、ホワイトリスト、Origin 検証
- HTTPS強制、Content Security Policy（CSP）
- 入力バリデーション、不正なリクエスト検証

### 4. 危険な部分の修正提案
- 危険なコードが見つかった場合のリファクタリング案
- シークレット管理への移行ガイド
- CORS の厳密な制限

### 5. デプロイ実行
- 本番環境のビルド実行
- ホスティングサービスへのデプロイ
- エンドポイントと API 健康チェック

### 6. 公開後検証
- 公開 URL でのアプリ機能確認
- 言語切替、会話フロー、スコア保存などの動作確認
- セキュリティ項目の再確認

## 判断ポイント

- ✓ デプロイ前に「開発用設定」が残っていないか
- ✓ API キーやシークレットが公開リポジトリに含まれていないか
- ✓ CORS や CSP が適切に設定されているか
- ✓ 外部入力処理が検証・制限されているか
- ✓ デプロイ先で本番用 URL と環境変数が正しく構成されているか

## 完了条件

- ✓ アプリが本番ホスティングで公開されている
- ✓ 主要 API が正しく応答している
- ✓ シークレットがソースコードに露出していない
- ✓ 危険なコードパターンが修正されている
- ✓ 本番用設定が有効になっている

- **バックエンド**：Cloudflare Workers、Node.js、Python など

## セキュリティ確認項目

### 環境変数・シークレット
- [ ] APIキーがソースコードに含まれていない
- [ ] `.env` ファイルが `.gitignore` に追加されている
- [ ] 本番用シークレットが環境管理システムに登録されている
- [ ] 開発環境と本番環境で異なるキーを使用している

### コード安全性
- [ ] `eval` や `new Function` のような危険な関数が使われていない
- [ ] 外部入力が適切にサニタイズされている
- [ ] SQL インジェクション対策が実装されている
- [ ] XSS 対策が実装されている

### API設定
- [ ] CORS が厳密に設定されている
- [ ] ホワイトリスト方式の Origin 検証がある
- [ ] レート制限が実装されている
- [ ] HTTPS が強制されている

### ヘッダー設定
- [ ] Content Security Policy（CSP）ヘッダーが設定されている
- [ ] HSTS（Strict-Transport-Security）ヘッダーが設定されている
- [ ] X-Content-Type-Options ヘッダーが設定されている
- [ ] X-Frame-Options ヘッダーが設定されている

### 外部依存関係
- [ ] npm audit で脆弱性がないことを確認
- [ ] 依存ライブラリが最新バージョンに更新されている
- [ ] 非公式・危険なサードパーティコードが含まれていない
